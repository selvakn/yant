# Data Model: Multiple Drawings Per Note

## Entities

### note_drawings (new table)

| Field        | Type    | Constraints                              | Description                          |
|--------------|---------|------------------------------------------|--------------------------------------|
| drawing_id   | TEXT    | NOT NULL, PK (composite with note_id)    | 8-char alphanumeric stable ID        |
| note_id      | INTEGER | NOT NULL, FK → notes(id) ON DELETE CASCADE | Parent note                         |
| display_name | TEXT    | NOT NULL, max 100 chars                  | User-provided label, mutable         |
| tool_type    | TEXT    | NOT NULL, CHECK IN ('tldraw','excalidraw') | Drawing tool used                  |
| created_at   | TEXT    | NOT NULL                                 | ISO 8601 timestamp                   |
| updated_at   | TEXT    | NOT NULL                                 | ISO 8601 timestamp                   |

**Primary Key**: (drawing_id, note_id)  
**Index**: idx_note_drawings_note ON note_drawings(note_id)

### Drawing Files (filesystem)

| Pattern | Example | Description |
|---------|---------|-------------|
| `<userID>/<slug>--<drawingID>.<tool>.json` | `42/my-note--abc12345.excalidraw.json` | New multi-drawing format |
| `<userID>/<slug>.<tool>.json` | `42/my-note.tldraw.json` | Legacy single-drawing format (read-only compat) |

### Drawing Marker (in markdown)

Syntax: `![[draw:<drawingID>]]`

Embedded inline in the note's markdown body. Determines render position in reader mode. Each drawing should have exactly one marker. Orphan markers (no matching file) render as a placeholder. Orphan files (no marker) are listed in the editor's drawing management UI but not rendered in reader mode.

## Relationships

```
notes (1) ──── (*) note_drawings
                     │
                     └── drawing files on disk (1:1 with note_drawings row)
                     └── marker in markdown body (1:1, position-based)
```

## State Transitions

### Drawing Lifecycle

```
[No Drawing] ──(Add drawing)──→ [Active]
                                    │
              ┌─(Delete drawing)────┘
              ↓
         [Deleted] (file removed, marker becomes orphan placeholder)
```

### Legacy Migration

```
[Legacy File] ──(first edit / add 2nd drawing)──→ [Migrated to new format]
    └── <slug>.tldraw.json                            └── <slug>--<id>.tldraw.json
                                                      └── note_drawings row inserted
                                                      └── ![[draw:<id>]] appended to markdown
```

## Validation Rules

- Drawing ID: 8 alphanumeric characters, generated by system
- Display name: 1–100 characters, plain text (alphanumeric, spaces, hyphens, underscores)
- Tool type: strictly "tldraw" or "excalidraw"
- Drawing ID + note_id must be unique (enforced by PK)
- Display name uniqueness within a note: soft warning, not hard constraint
